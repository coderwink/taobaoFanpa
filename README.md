# 淘宝会员页视觉自动化采集系统（AI + 无障碍 + 视觉Agent）技术路线

## 一、项目定位

本项目目标：

> 自动化获取淘宝店铺会员页、小部件页、swiper 卡片等动态内容。

由于淘宝当前：

- 大量页面 Native 化
- 小部件 DSL 化
- WebView 弱化
- DOM 不完整
- Swiper 离屏渲染
- 懒加载
- 原生容器嵌套

传统：

- H5爬虫
- JS注入
- DOM解析

已经越来越难获取完整数据。

因此：

# 未来方向是：

## 视觉驱动自动化（Visual Automation）

即：

```text
AI看屏幕
↓
自动操作APP
↓
自动截图
↓
OCR识别
↓
AI理解页面
↓
结构化输出
```

# 三、为什么传统方案失效

## 1. DOM 不完整

淘宝会员页：

很多内容：

- 不在 WebView
- 不在 HTML
- 不在 JS Runtime

而是：

- Native
- Flutter
- Weex
- Lynx
- 自定义渲染引擎

因此：

```js
document.querySelector()
```

很多时候获取不到。

---

## 2. Swiper 离屏销毁

淘宝 swiper：

通常：

- 仅渲染当前页
- 左右预加载
- 离屏销毁

所以：

你无法一次性获取完整数据。

---

## 3. Canvas 化

部分活动页：

直接：

- Canvas绘制
- OpenGL绘制

OCR 反而比 DOM 更有效。

# 六、Swiper 自动遍历方案

这是整个系统核心。

---

## 核心思路

不要：

# “读取 swiper 数据”

而是：

# “遍历 swiper”

---

## 实现流程

```text
进入会员页
↓
截图第一页
↓
记录图像hash
↓
向左滑动
↓
等待动画结束
↓
截图第二页
↓
对比hash
↓
不同 = 新页面
↓
继续滑动
↓
重复则停止
```

---
